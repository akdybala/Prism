# Code-Query Pair Corpus

Generated by:

```powershell
.\.benchmark_venv\Scripts\python.exe build_code_query_pairs.py
```

The corpus contains 8 realistic, deliberately ambiguous queries for each
function. Every function family covers all six operations:

`explain`, `debug`, `optimize`, `review`, `generate`, and `refactor`.

Queries use PR-description, issue-comment, code-review, maintenance,
investigation, and mixed-request styles. Labels include query domain, operation,
concerns, ambiguity, provenance, license, repository revision, and a
family-level train/validation/test split.

Files:

- `code_query_pairs.csv`: tabular training/export format.
- `code_query_pairs.jsonl`: lossless line-oriented format.
- `dataset_report.json`: counts and coverage.
- `repository_audit.json`: accepted/rejected repositories and file counts.

All variants of one function are assigned to the same split, preventing the
same code from leaking across train and evaluation sets.

These records can train operation/domain/concern models or provide the request
side of an XGBoost/LightGBM router. To train the repository's candidate-quality
router, join each `pair_id` with one observed row per candidate containing
`success`, quality, latency, token usage, and cost. A `(code, query)` pair alone
does not identify which candidate model succeeds.
